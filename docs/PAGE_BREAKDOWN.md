# 📄 页面与功能模块拆解文档

本文件详细列出餐馆点餐系统的各个页面与其对应的功能，便于开发阶段逐步实现和测试。

---

## 👤 用户端页面与功能

### `/menu`
- 显示所有菜品（名称 / 图片 / 描述 / 价格）
- 支持按分类筛选（如主食、饮料、甜点）
- 点击加入购物车
- 显示库存状态（是否可点）

### `/cart`
- 展示用户已选菜品列表
  - 每项显示名称 / 数量 / 小计 / 删除按钮
- 修改数量（+ / -）
- 显示总价
- 跳转至结账页 `/checkout`

### `/checkout`
- 显示订单摘要（所有菜品 + 总价）
- 填写或确认联系方式（如手机号）
- 使用 Stripe 生成支付链接并跳转
- 支付完成后跳转到 `/order-success`

### `/order-success`
- 展示“支付成功”页面
- 简要订单摘要 + 等待时间提示

---

## 🧑‍💼 管理端页面与功能（需登录）

### `/admin/login`
- 管理员登录表单（email + password）
- 验证成功后跳转至 `/admin/menu`

### `/admin/menu`
- 展示所有菜品
- 添加新菜品（表单）
- 编辑 / 删除菜品
- 设置菜品是否上架 / 可点状态

### `/admin/orders`
- 查看所有订单（按时间倒序）
- 每个订单展示用户信息、下单时间、订单内容、总价、支付状态、当前状态
- 修改订单状态（制作中 / 已完成 / 已取消）

---

## 🔗 系统通用功能模块

### 权限控制
- 管理端路由需登录验证（JWT + middleware）
- 用户下单无需登录，但需填写手机号（可用于后续订单查询）

### 数据持久化
- 所有数据存储在 Neon PostgreSQL 数据库
- 使用 Prisma 管理数据模型、迁移与访问

### 支付系统（Stripe）
- Stripe Checkout 创建订单支付链接
- Stripe Webhook 接收支付结果并更新订单状态

---

## ⛏️ 后续可扩展页面（非 MVP 阶段）

### `/track-order/:orderId`
- 用户输入订单号或扫码查看当前状态

### `/admin/analytics`
- 销售统计图表（本周 / 本月订单总额、菜品热度等）
